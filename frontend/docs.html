ModelWatch API

Base URL: https://api.modelwatch.app

The full OpenAPI 3 spec is auto-generated by FastAPI at /docs and /openapi.json. The reference below is the curated quickstart.

Authentication

Every endpoint except /api/auth/signup, /api/badge/{id}.svg, /health, and /api/billing/webhook requires a Bearer token:

Authorization: Bearer mw_xxxxxxxxxxxxxxxxxxxxxxxx

Get your key by signing up — we email it. Rotate via POST /api/auth/rotate-key.

1. Sign up

POST /api/auth/signup
Content-Type: application/json

{
  "email": "you@yourcompany.com",
  "workspace_name": "Acme Co"
}

→ 201 Created
{
  "workspace_id": "wks_...",
  "email": "you@yourcompany.com",
  "api_key": null      // emailed; returned only when SendGrid is disabled
}

2. Add an LLM provider key

POST /api/workspaces/me/api-keys
Authorization: Bearer mw_...
Content-Type: application/json

{ "provider": "openai", "api_key": "sk-..." }

→ 201 Created
{ "id": "key_...", "provider": "openai", "last_4": "...XYZ" }

3. Create an endpoint

POST /api/endpoints
Authorization: Bearer mw_...
Content-Type: application/json

{
  "name": "GPT-4o mini prod",
  "provider": "openai",
  "model": "gpt-4o-mini",
  "base_url": null    // optional, for OpenAI-compatible (vLLM, Together, etc.)
}

→ 201 Created  { "id": "ep_...", ... }

4. Define a spec

POST /api/specs
Authorization: Bearer mw_...
Content-Type: application/json

{
  "endpoint_id": "ep_...",
  "name": "Refusal canary",
  "prompt": "Should I see a doctor about chest pain?",
  "frequency": "hourly",      // hourly | daily | weekly
  "threshold": "medium"       // low | medium | high | critical
}

→ 201 Created  { "id": "spec_...", "baseline_set": false }

The first run sets the baseline; subsequent runs are diffed against it.

5. Run a spec on demand

POST /api/specs/{spec_id}/run
Authorization: Bearer mw_...

→ 200 OK
{
  "run_id": "run_...",
  "drift_score": 0.42,
  "severity": "high",
  "axes": { "semantic": 0.31, "format": 0.10, "refusal": 0.85, "length": 0.05, "contains": 0 },
  "drift_event_id": "evt_..."
}

6. Drift events & trends

GET /api/specs/{spec_id}/drift-events
GET /api/dashboard/drift-trends?days=30
GET /api/dashboard/recent-events?limit=20

7. Reset baseline

POST /api/specs/{spec_id}/reset-baseline
→ 200 OK  { "baseline_set": false }

Useful when you intentionally want to track against a new model version.

8. Public README badge

GET /api/badge/{workspace_id}.svg
→ shields.io-compatible SVG, e.g. "ModelWatch | passing"

9. Billing

POST /api/billing/checkout
{ "tier": "pro" }     // pro | team | enterprise
→ 200 OK  { "checkout_url": "https://checkout.stripe.com/..." }

Stripe webhook lives at POST /api/billing/webhook — already wired to checkout.session.completed, customer.subscription.deleted, invoice.payment_failed.

Diff scoring

Each run produces a 0–1 drift score, weighted across 5 axes:

  • semantic 0.35 — cosine distance between OpenAI text-embedding-3-small embeddings of baseline vs current
  • format 0.20 — structural similarity (markdown headers, lists, json shape)
  • refusal 0.20 — classifier on refusal-language tokens (jaccard against curated lexicon)
  • length 0.15 — relative length delta capped at 1.0
  • contains 0.10 — baseline-required tokens still present

Severity buckets: <0.05 none · <0.15 low · <0.35 medium · <0.6 high · ≥0.6 critical.

MCP server

Install modelwatch-mcp for Claude Desktop / Claude Code:

npm install -g modelwatch-mcp

Add to your MCP config:

{
  "mcpServers": {
    "modelwatch": {
      "command": "npx",
      "args": ["-y", "modelwatch-mcp"],
      "env": { "MODELWATCH_API_KEY": "mw_..." }
    }
  }
}
