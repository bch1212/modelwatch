## Show HN

**Title:**
> Show HN: ModelWatch — tells you when OpenAI / Anthropic silently change a model

**Body:**

I built ModelWatch because my own production agents kept breaking on Tuesday
afternoons for no apparent reason. Turned out the providers were quietly
re-tuning their hosted models — same model name, slightly different behavior —
and my CI didn't run on Tuesdays.

The pitch: continuous behavioral monitoring of *their* model.

You define a behavioral spec (input prompt + expected behavior). ModelWatch
replays it against the live LLM endpoint on a schedule (hourly / daily /
weekly), scores the drift against your baseline across 5 axes (semantic,
format, refusal, length, contains), and alerts when the score crosses your
threshold.

The wedge vs. DeepEval / Braintrust / LangSmith: those tools run when *you*
trigger them (CI). ModelWatch runs continuously against the live provider.

Bring-your-own OpenAI / Anthropic / Gemini / Groq key — zero LLM markup. Free
tier (5 specs, 500 runs/mo) is enough for one product's canary suite.

There's also `modelwatch-mcp` on npm (and the Anthropic MCP Registry) so you
can drive the workspace from Claude Desktop / Claude Code.

A weekly public Drift Report ships every Monday — auto-generated by running
the same canary specs against GPT-4o / Claude / Gemini / Llama and posting the
drift table.

- Live: https://modelwatch.app
- Source: https://github.com/bch1212/modelwatch
- MCP: https://www.npmjs.com/package/modelwatch-mcp

Curious what behavioral changes you've seen in production from the major
providers — happy to add specific canary prompts to the public report.

## r/mcp

**Title:** [Tool] modelwatch-mcp — drift monitoring for the LLMs your agents call

**Body:**

If your agent calls a hosted LLM, you're at the mercy of the provider's silent
model updates. ModelWatch is a continuous drift monitor that replays your
behavioral specs against the live endpoint on a schedule and alerts when
output shifts vs. baseline.

The MCP wrapper: 9 tools — `list_endpoints`, `create_endpoint`, `list_specs`,
`create_spec`, `run_spec`, `reset_baseline`, `get_drift_events`,
`get_spec_history`, `get_health`. stdio transport.

Listed in the Anthropic MCP Registry as `io.github.bch1212/modelwatch`.
